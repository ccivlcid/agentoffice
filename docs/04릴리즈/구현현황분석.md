## v1.2.2 — 전체 미구현

> v1.2.2의 모든 핵심 기능이 코드에 반영되지 않았습니다.

### 1. Interrupt-Inject 워크플로우 (POST /api/tasks/:id/inject)

- **문서 기술**: 일시정지된(paused) 태스크에 프롬프트를 주입하여 실행 재개하는 워크플로우
- **현재 상태**: `execution-control.ts` 파일 없음, inject 라우트 미등록, `server/modules/workflow/core/` 디렉토리 자체 없음
- **UI/UX 기술 내용**:

  **TerminalPanel에 Interrupt/Inject 패널 추가**

  현재 TerminalPanel은 **읽기 전용** (터미널 로그 스트리밍 + 회의록 탭). 아래 UI가 추가되어야 함:

  ```
  ┌─────────────────────────────────────────────────┐
  │ [터미널] [회의록]                    FOLLOW  ✕  │
  │                                                  │
  │  ┌─ 터미널 로그 출력 ──────────────────────────┐ │
  │  │ > Installing dependencies...                │ │
  │  │ > Running build...                          │ │
  │  │ > ▌ (paused)                                │ │
  │  └─────────────────────────────────────────────┘ │
  │                                                  │
  │  ┌─ Interrupt/Inject 패널 ─────────────────────┐ │
  │  │ ┌─────────────────────────────────────────┐ │ │
  │  │ │  프롬프트 입력...                       │ │ │
  │  │ └─────────────────────────────────────────┘ │ │
  │  │                                             │ │
  │  │  [일시정지만]  [주입 후 재개]  [바로 재개]  │ │
  │  └─────────────────────────────────────────────┘ │
  │                                                  │
  │  agent-name · claude · ●                         │
  └─────────────────────────────────────────────────┘
  ```

  - **일시정지만** (Pause only): 태스크를 일시정지 상태로 유지
  - **주입 후 재개** (Inject + Resume): 입력한 프롬프트를 큐에 넣고 즉시 재개
  - **바로 재개** (Resume only): 프롬프트 없이 재개
  - 일시정지 상태가 아닌 태스크에서는 패널 비활성화
  - 프롬프트 해시 감사 로그 기록

  **백엔드 보안 요건**:
  - `x-task-interrupt-token` 헤더 또는 body `interrupt_token`으로 제어 토큰 검증
  - Cookie 인증 시 CSRF 토큰 필수
  - Terminal API가 interrupt proof payload (`session_id`, `control_token`, `requires_csrf`) 반환

### 2. 에이전트별 CLI 모델 오버라이드

- **문서 기술**: 각 에이전트에 개별 CLI 메인 모델과 추론 레벨 설정 가능
- **현재 상태**: DB 스키마에 `cli_model`/`cli_reasoning_level` 컬럼 없음. 현재 모델 설정은 전역 `CompanySettings.providerModelConfig`에서만 관리
- **UI/UX 기술 내용**:

  **AgentDetail CLI 편집기 확장** (`AgentDetailCliEditor.tsx`)

  현재 AgentDetailCliEditor는 CLI 프로바이더 `<select>` + OAuth 계정 선택만 지원.
  아래 필드가 추가되어야 함:

  ```
  ┌─ CLI 설정 ──────────────────────────────────┐
  │ Provider:  [claude ▼]                        │
  │ Model:     [claude-sonnet-4-6 ▼] (default)   │  ← 새로 추가
  │ Reasoning: [medium ▼]                        │  ← codex 전용, 새로 추가
  │                                              │
  │           [저장]  [취소]                      │
  └──────────────────────────────────────────────┘
  ```

  - **Model 드롭다운**: 선택한 CLI 프로바이더의 모델 목록에서 선택. "default"는 전역 Settings 값 사용
  - **Reasoning 드롭다운**: `codex` 프로바이더 선택 시에만 표시. `low` / `medium` / `high` / `xhigh` + `default`
  - 서브에이전트 모델은 이 설정에 영향받지 않음 (전역 `subModel` 유지)
  - **런타임 전파 범위**: task run, resume, one-shot, direct spawn, delegated subtask batch, cross-department cooperation — 6개 실행 진입점 모두

### 3. Planning-lead noTools fast path

- **문서 기술**: 팀장 미팅/의사결정 원샷에서 `noTools: true` 설정으로 도구 호출 없이 빠른 텍스트 생성
- **현재 상태**: `server/modules/workflow/` 전체에서 `noTools` 옵션 미사용
- **적용 대상**:
  - 계획 승인 미팅 (planned-approval) 원샷
  - 리뷰 합의 미팅 (review-consensus) 원샷
  - 프로젝트 리뷰 계획 통합 (project_review_ready)
  - 리뷰 라운드 계획 통합 (review_round_pick)
  - 최종 보고서 아카이브 생성
- **UI 영향**: 사용자에게 직접 보이지 않으나 미팅/의사결정 속도 개선

### 4. scripts/qa/interrupt-inject-http-smoke.mjs

- **문서 기술**: Interrupt-Inject 기능의 HTTP 스모크 테스트 스크립트
- **현재 상태**: 없음. 기능 1번(Interrupt-Inject) 구현 후 작성 대상

### 3. 메신저 멀티-토큰 라우트 격리

- **문서 기술**: 같은 채널(예: Telegram)에 여러 봇 토큰 사용 시, `targetId` 겹침 문제를 토큰 힌트 키(`<channel>#<tokenKey>`)로 해결
- **현재 상태**: 단일 offset 필드, 토큰 힌트 처리 없음
- **UI/UX 기술 내용**:

  **문제 시나리오**:
  ```
  Bot A (token-AAA) → Chat ID 12345
  Bot B (token-BBB) → Chat ID 12345 (같은 그룹)
  → 현재: 두 봇의 메시지가 뒤섞여 라우팅 오류
  → 목표: 각 봇이 자기 세션으로만 응답
  ```

  - Gateway 설정에서 같은 채널에 다른 토큰으로 여러 세션 등록 가능
  - 내부적으로 `telegram#abc123` 형식 키로 라우트 구분 (사용자에게 토큰 노출 안 함)
  - Telegram receiver가 per-token offset 관리 → 크로스 토큰 poll 충돌 방지

### 4. video_preprod Remotion 렌더 오케스트레이션

- **문서 기술**: Remotion 기반 실제 MP4 출력, `[VIDEO_FINAL_RENDER]` 서브태스크를 기획 단계에서 시드하고 다른 서브태스크 완료 후 위임
- **현재 상태**: 코드베이스에 `VIDEO_FINAL_RENDER`, `Remotion` 관련 코드 전무
- **UI/UX 기술 내용**:
  - TaskBoard에서 `[VIDEO_FINAL_RENDER]` 서브태스크가 별도 표시
  - 비렌더 서브태스크 전체 완료 전까지 렌더 서브태스크는 대기 상태
  - 리뷰 회의 라우팅이 블록되면 렌더 위임을 큐에 넣고 자동 재개
  - 중복 렌더 서브태스크 생성 방지 + stale 위임 링크 정리

### 5. Report/PPT 이중 출력 정책

- 아이콘: `lucide-react` + 직접 SVG (CLI 로고)
- 다국어: `useI18n()` + `t({ ko, en})` 2개 언어
