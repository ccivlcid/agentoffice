📋 SPEC: 에이전트 회의실
1. 개요
웹 대시보드 안에서 사용자가 AI 에이전트들과 화상회의하는 인터페이스.

사용자: 웹캠으로 참여 (중앙 화면)
AI 에이전트: 아바타 + 음성으로 참여
멘션(@이름)으로 특정 에이전트 지정 가능
우측에 실시간 회의록 자동 기록
2. 기술 스택
항목	기술
프레임워크	Next.js 14+ (App Router)
UI	Tailwind CSS + shadcn/ui
카메라	navigator.mediaDevices.getUserMedia
음성입력(STT)	OpenAI Whisper API 또는 브라우저 Web Speech API
음성출력(TTS)	브라우저 SpeechSynthesis (무료) 또는 ElevenLabs API
AI 백엔드	Anthropic Claude API / OpenAI API / Cerebras (무료)
상태관리	React useState + useRef

5. 핵심 기능
5-1. 에이전트 초대/퇴장
좌측 패널에 에이전트 목록
클릭하면 회의 그리드에 추가 (아바타 표시)
다시 클릭하면 퇴장
5-2. 메시지 전송 (텍스트 + 음성)
텍스트 입력 또는 🎤 버튼으로 음성 입력
음성: MediaRecorder → Whisper API로 STT → 자동 전송
@에이전트이름으로 특정 에이전트 지정 (없으면 마지막 참여자가 응답)
5-3. 에이전트 응답
서버에서 에이전트 시스템 프롬프트 + 대화 히스토리로 LLM 호출
응답 텍스트를 TTS로 음성 재생
해당 에이전트 아바타에 "speaking" 애니메이션
5-4. 아바타 상태
상태	표시
idle	회색 테두리, 이모지
thinking	노랑 점멸 (API 호출 중)
speaking	초록 glow + pulse
5-5. 회의록
우측 패널에 자동 기록
발언자 이름 + 이모지 + 타임스탬프
스크롤 자동
6. API 설계
POST /api/meeting/chat
// Request
{
  message: string          // 사용자 메시지
  agentId?: string         // 지정 에이전트 (없으면 자동 감지)
  participants: string[]   // 현재 참여 에이전트 ID 목록
  history: {               // 최근 대화 (10개)
    role: 'user' | 'assistant'
    content: string
    agent?: string
  }[]
}

// Response
{
  agentId: string          // 응답한 에이전트
  message: string          // 응답 텍스트
  timestamp: string
}
서버 로직 (route.ts)
// 1. 메시지에서 @멘션 감지 → 해당 에이전트 선택
// 2. 에이전트 시스템 프롬프트 로드
// 3. (선택) 에이전트별 실시간 데이터 fetch → 프롬프트에 첨부
// 4. LLM API 호출 (Claude / GPT / Cerebras)
// 5. 응답 반환
LLM 선택 가이드:
| 옵션 | 비용 | 품질 | 추천 |
|------|------|------|------|
| Cerebras (qwen-3-235b) | 무료 | 좋음 | ✅ 입문자 추천 |
| Claude Haiku | 저렴 | 매우 좋음 | 비용 여유 있으면 |
| GPT-4o-mini | 저렴 | 매우 좋음 | OpenAI 선호 시 |
| Claude Sonnet | 중간 | 최고 | 프로덕션 |

7. TTS (음성 출력) 옵션
옵션 A: 브라우저 내장 (무료, 간단)
const speak = (text: string) => {
  const utter = new SpeechSynthesisUtterance(text)
  utter.lang = 'ko-KR'
  utter.rate = 1.1
  speechSynthesis.speak(utter)
}
옵션 B: ElevenLabs API (유료, 고품질)
// /api/meeting/tts (POST)
// body: { text, voiceId }
// ElevenLabs API → audio buffer → 클라이언트에서 재생
에이전트마다 다른 voiceId 지정 → 개성 있는 목소리

8. STT (음성 입력) 옵션
옵션 A: 브라우저 Web Speech API (무료)
const recognition = new webkitSpeechRecognition()
recognition.lang = 'ko-KR'
recognition.continuous = false
recognition.onresult = (e) => setInput(e.results[0][0].transcript)

// MediaRecorder → audio blob → /api/stt (POST)
