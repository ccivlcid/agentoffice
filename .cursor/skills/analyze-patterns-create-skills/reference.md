# 패턴 유형 참고

스킬로 옮길 때 참고할 개발 패턴 유형.

## 규칙형 패턴

- **필수/금지**: "~해야 한다", "~하면 안 된다" → 체크리스트 또는 단일 문장.
- **값/설정**: 상수, 기본값, 옵션 → "기본은 X, Y일 때는 Z" 형태로 한 곳에 정리.
- **네이밍**: 파일/변수/함수/API 이름 규칙 → 예시 1~2개 포함.

## 워크플로우 패턴

- **순차 단계**: A → B → C → 검증. 각 단계를 번호/체크리스트로.
- **조건 분기**: "신규일 때 / 수정일 때" 등 → "Determine ... → If X follow A, if Y follow B".
- **피드백 루프**: 수정 후 검증 필수인 경우 → "수정 → validate → 실패 시 수정 반복" 명시.

## 출력/템플릿 패턴

- **포맷**: 커밋 메시지, API 응답, 로그 형식 → 예시 1~2개를 Examples에.
- **파일 구조**: 디렉터리/파일 배치 → 트리 또는 "다음 파일을 만든다" 목록.

## 도메인별로 자주 나오는 것

- **API/백엔드**: 라우트 구조, 스키마 검증, 에러 코드, 로깅 레벨.
- **프론트**: 컴포넌트 분리, 상태 위치, 훅 사용 규칙.
- **DB**: 마이그레이션 순서, 네이밍, 인덱스 규칙.
- **공통**: 환경 변수, 시크릿, 테스트 위치/네이밍.

이 유형에 맞춰 패턴을 분류하면 Instructions/Examples 구성이 수월하다.
